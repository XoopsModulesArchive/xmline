聚合新闻(XMLine)说明文件



XMLine是做什么用的

XMLine 用于采集显示XML内容，包括各类RSS和ATOM.
可以看作是XOOPSHEADLINE的加强版.

    功能:
  1. 支持的类型: RSSs, ATOM
  2. 可用的采集方法: CURL, fsockopen, fopen.
  3. 多类别
  4. 多内码转换
  5. 可开放用户订制或提交
  6. Table-free模板
  7. 无限制复制区块
  8. 可自动更新(CRONJOB)

使用聚合新闻模块时，请务必详细阅读本说明文档。

有些空间不能运行该模块，详情请看管理区相关测试。

一、安装

把从XOOPS.org.cn下载的文件解压缩，得到xoops/modules/xmline目录，然后进入modules目录，把xmline目录和xconv目录上传到你XOOPS的modules下，再进入“管理区=>系统管理=>模块管理”安装“聚合新闻”模块和“Xoops Iconv”模块（图标带有“xconv”文字，）。这样就完成安装了。

关于Xoops Iconv模块

Xoops Iconv是转换内码的一个模块，比PHP的iconv函数效率更高。如果不想安装，就可以不用上传xconv目录和安装“Xoops Iconv”模块。虽然这个模块不是必须，但是我们非常推荐安装这个模块。

二、使用

分类管理：进入“管理区=>聚合新闻（把鼠标放在左栏带有“XMLINE”文字的图标）=>类别管理”，来进行分类的管理。

添加网址：进入“管理区=>聚合新闻（把鼠标放在左栏带有“XMLINE”文字的图标）=>站点管理”来添加网址。

设置项：

RSS 网址：你想要读取的RSS地址。

主题: 一般用几个字符标明，不要太长（所有首页显示的网摘都会显示为“[主题]标题”，因此主题不要太长，否则严重影响显示效果）

网址：RSS所属网页地址, 可不填。

描述：是对“主题”的进一步说明，可以详细一点介绍，可不填。

分类：你添加的这个网摘属于什么分类。

站点logo：没有可以不用。

顺序: 设置网站显示顺序

Online: 如果填“1”，这个网站将会在前台出现，反之如果添“0”这个网站将不会在前台出现。

最大条目: 限制要存取的最大新闻条数

网页charset编码: 如果留空, 则自动检查待采集页面的charset；如果强制不做内码转换，输入 null

网页过渡编码: 从繁体UTF-8到简体GB, 填入BIG5; 从简体UTF-8到繁体BIG5，填入GB2312。

刷新频率: 填上你要更新数据的频率，单位为分钟

　

关于Cron:

Cron是UNIX/Linux系统下一个定时执行的指令，利用它可以定时触发聚合新闻的更新，也就是定时更新你的文摘，免去手动更新的麻烦，而且程序在后台运行，你感受不到速度的影响。（也可也以采用虚拟Cron, Pseudo-Cron，由访问用户触发。那么碰巧触发更新的那个用户，页面载入会特别慢）。

再添加Cron之前，你必须确定：

你的空间拥有Cpanel，向你的空间服务商询问是否拥有Cpanel。

如果你的空间没有Cpanel或无法使用Cron，可以告知 http://xoops.org.cn，定时触发你的网络文摘更新。

方法

1.
检查文摘更新接口文件, 在xoops根目录下的cache/xmline/xmline.php。该文件在模块安装时自动生成，可进入文摘模块的管理区检查该文件是否存在。如果不存在，可以"重新生成该文件"。

2.
然后在Cpanel中添加CronJOB

需要注明的是，在Command to run 输入框中，需要填入调用Cron脚本的绝对路径，也就是你的cron.php所在的绝对路径。

例如：/usr/local/bin/php /用户XOOPS绝对路径/xoops/cache/xmline/xmline.php

你可以向你的空间服务商所取绝对路径地址。

3.
如果你的空间没有Cron功能,你可以请其他有cron功能的服务器或站点从外部触发。
调用文件为 http://你的xoops网址/cache/xmline/xmline.php

4.
如果路径http://你的xoops网址/cache/xmline/xmline.php不可用，请直接调用http://你的xoops网址/modules/xmline/update.php
